IMPORTANTE — por qué no se eliminaron del DOM:
El script (más abajo) escribe directamente sobre estos elementos
SIN validar que existan:
  - document.getElementById('utmActualMes').innerText = ...
  - document.getElementById('utmActualVal').innerText = ...
  - document.getElementById('utmTicker').classList.remove('hidden')
  - document.getElementById('utmTicker').style.display = 'flex'
  - document.getElementById('activeCasoNombreHeader').innerText = email
    (en el flujo de actualización del badge de correo)
Eliminar esos elementos del HTML habría lanzado TypeError
("Cannot set property ... of null") y roto la ejecución del
script para abajo — el mismo patrón de falla que causó el bug de
la welcome screen en una versión anterior (ver CAMBIOS v112l).

Por eso se dejaron en el DOM pero ocultos visualmente. El script
sigue corriendo exactamente igual; estos elementos simplemente no
se muestran en pantalla.

NO SE TOCÓ: ninguna lógica de cálculo, ningún otro onclick/handler,
ni el comportamiento de openSidebar()/openCasosManagerModal()/
abrirUserProfile() — solo se reordenó/ocultó HTML.
