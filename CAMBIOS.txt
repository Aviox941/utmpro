PROPÓSITO DE ESTA VERSIÓN

FIX — IDB backup/restore ampliado a TODAS las claves críticas
--------------------------------------------------------------
PROBLEMA ANTERIOR (v72): IDB solo respaldaba pension_utm_auth. Si Android
limpiaba todo el localStorage (no solo el auth), el fallback del timeout
también fallaba porque getCasosIndex() devolvía [] y pension_utm_last_caso
era null — resultado: welcome screen vacía aunque hubiera sesión válida.

CLAVES AHORA RESPALDADAS EN IDB:
  - pension_utm_auth              (token de sesión Supabase)
  - pension_utm_last_caso         (último caso activo)
  - pension_utm_casos_index_v1    (índice de todos los casos)
  - pension_utm_caso_v1_<id>      (datos de cada caso individual)

IMPLEMENTACIÓN:
1. IDB restore al arranque (ya existía):
   Ampliado con getAllKeys() — detecta y restaura TODAS las claves
   respaldadas de una vez, incluyendo datos de cada caso.
   Solo corre si localStorage está vacío (pension_utm_auth ausente).

2. Storage adapter _sbStorage (ya existía):
   Ampliado con _idbShouldBackup() — respalda auth + last_caso +
   casos_index + caso_v1_* automáticamente en cada setItem.

3. Monkey-patch localStorage (NUEVO):
   Dado que saveCasosIndex(), saveCurrentCasoNow() y otras funciones
   escriben directo a localStorage (no pasan por _sbStorage), se
   intercepta localStorage.setItem/removeItem para que las claves
   pension_utm_* también se respalden en IDB automáticamente.

FIX ANTERIOR (v72b, incluido):
   sbLoadCasos() timeout 10s + fallback a localStorage si Supabase no responde.
