FIX — IDs no-UUID generados desde welcome screen nunca se borraban de Supabase
------------------------------------------------------------------------

PROBLEMA RAÍZ: dos bugs relacionados causaban que casos creados desde la
welcome screen reaparecieran al reiniciar aunque el usuario los eliminara.

FIX A — crearCasoDesdeWelcome() usaba siempre 'caso_' + Date.now() (export.js):
  La función tenía el ID hardcodeado como string no-UUID, a diferencia de
  crearCaso() que ya usaba crypto.randomUUID con fallback. Corregido para
  usar exactamente la misma lógica: crypto.randomUUID() si está disponible,
  fallback 'caso_' + Date.now() + random solo si no hay soporte.

FIX B — _sbDeleteCasoCore no limpiaba pending_deletes para IDs no-UUID (index.html):
  El guard que omite el DELETE a Supabase cuando el id no es UUID hacía
  return inmediato sin limpiar pending_deletes. El ID quedaba en la lista
  permanentemente y en cada reinicio se reintentaba el delete sin éxito.
  Ahora limpia pending_deletes antes del return, igual que hace el path
  exitoso del DELETE real.
